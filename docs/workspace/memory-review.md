# Memory Review

> **Candidate review is the gate between temporary AI output and durable project knowledge.**

## Candidate sources

- manual capture;
- normalized agent proposal;
- future import workflows.

## Validation

Before persistence, candidates are checked for:

- required structure;
- length;
- duplicate content;
- workspace scope;
- credential-like patterns;
- complete source-file leakage;
- transient or unsupported claims.

## User actions

The user may:

- approve;
- edit and approve;
- reject;
- leave pending;
- archive an existing memory;
- mark an existing memory outdated.

## Approval

Approval creates or activates a durable memory. The original candidate provenance remains available for audit.

## Core rule

> **Nothing becomes durable knowledge without a visible user decision.**
