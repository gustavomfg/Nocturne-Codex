# Sessions

> **A session is a bounded period of work with a clear objective and execution history.**

## Purpose

Sessions group related turns, commands, tools, artifacts and decisions so the user can understand what happened during a task.

## Suggested fields

- workspace and conversation;
- objective;
- start and finish time;
- mode;
- status;
- selected provider and model;
- linked executions;
- summary and unresolved items.

## Relationship to execution

A session may contain multiple executions, including retries or a Review followed by an approved Build.

## Relationship to memory

Sessions may produce candidates, but they never become memory automatically.

## Core rule

> **Sessions explain the work. Memories preserve the knowledge.**
