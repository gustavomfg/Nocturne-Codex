# Nocturne Codex v0.8.0-beta

Release date: July 16, 2026

`v0.8.0-beta` is a stability-focused preview for product validation. It strengthens the real Codex contract, persistence rollback, update lifecycle, long-session rendering, package size, and release provenance without claiming the maturity of a stable release or weakening the local trust boundary.

## Highlights

- Natural Build, Review, and Docs mode changes inside the same conversation.
- Codex CLI `0.144.5` verified through a real authenticated App Server contract smoke.
- Explicit compatibility states distinguish unsupported, minimum-compatible-unverified, and verified Codex CLI versions.
- Consent-based updates protected against duplicate checks, prompts, downloads, and installations.
- Atomic workspace context writes and asynchronous filesystem discovery.
- Transactional backup restore with duplicate-ID rejection and full rollback on malformed data.
- SQLite schema 7 indexes for conversation, workspace, artifact, and suggestion-decision navigation.
- Native offscreen rendering for long conversations and bounded streaming, diff, activity, and file state.
- Electron package restricted to the supported `pt-BR` and `en-US` locales.

## Compatibility

- Runtime: Node.js 24.18 or newer in the Node 24 LTS line, with npm 11.
- Minimum Codex CLI: `0.144.0`.
- Verified Codex CLI: `0.144.1` and `0.144.5`.
- Public beta artifacts: Linux x64 AppImage and tarball. Windows and macOS builds passed CI validation but are not distributed in this beta.
- Git remains required for repository workflows; Pandoc is optional for advanced document exports.

## Validation

The current source changes passed locally:

- TypeScript type checking;
- ESLint and design-system validation with zero warnings;
- 89 unit and integration tests, including a 25,000-message restore budget, future-schema rejection, credential redaction, lifecycle failures, and the schema 6 → 7 upgrade path;
- 24 deterministic renderer scenarios and four visual references;
- production renderer, main, and preload builds;
- a real authenticated Codex CLI `0.144.5` contract smoke;
- a packaged Linux runtime smoke covering SQLite, preload, sandboxing, context isolation, permissions, and external-navigation denial.

The published beta candidate separately recorded npm audit with zero known vulnerabilities; that network-dependent check is not implied by the local validation result above.

Stable publication additionally requires the protected workflow to repeat the complete source suite, package all three operating systems, smoke each signed package, verify Developer ID/notarization, Authenticode, GPG checksums, update metadata, and the exact-commit Codex smoke before GitHub Release creation.

## Upgrade and recovery

The database migrates automatically from the schema used by `v0.7.0-beta`. Conversations and messages are preserved while navigation indexes are added transactionally. Backup restore creates a recovery snapshot before replacement and rolls back all changes if any row violates the persistence contract.

## Known limitations

- The Codex App Server API remains experimental; unverified CLI versions are not promoted as supported.
- DOCX and PDF export quality depends on the locally installed Pandoc toolchain.
- The published beta artifacts are unsigned Linux previews and must not be treated as stable distribution artifacts.
- Stable artifacts cannot be published without the external Apple, Windows, and GPG signing identities configured in the protected GitHub environment.

## Linux installation

```bash
chmod +x Nocturne.Codex-Linux-0.8.0-beta.AppImage
./Nocturne.Codex-Linux-0.8.0-beta.AppImage
```

Nocturne Codex uses the locally installed Codex CLI authentication. It does not bundle credentials or copy user project data into a separate application service.
