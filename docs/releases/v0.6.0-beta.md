# Nocturne Codex v0.6.0-beta

Release date: July 11, 2026

`v0.6.0-beta` is the first release in which Nocturne Codex is treated as a cohesive desktop engineering product rather than a collection of agent features. It focuses on observable AI work, deliberate developer control, local persistence, security boundaries, and a calmer interface for long development sessions.

## Highlights

- Build, Review, and Docs interaction modes.
- Persistent conversations backed by SQLite and resumable Codex threads.
- Live agent activity, plans, approvals, changed files, and diffs.
- Structured review suggestions with severity, risk, complexity, affected files, and proposed changes.
- Explained Project Health estimates derived from open suggestions.
- Per-workspace memory and detected project context under `.nocturne/`.
- Artifact preservation, internal preview, Markdown saving, and optional Pandoc exports.
- Selective Git commits with exact path handling for Unicode and renames.
- Responsive desktop layout, accessible dialogs, keyboard navigation, and reduced-motion support.

## Security and reliability

- Electron renderer isolation with sandboxing and no Node.js integration.
- Typed preload surface and schema-validated IPC payloads.
- IPC sender and main-frame verification for privileged operations.
- Workspace path confinement, including traversal and symbolic-link protection.
- Read-only enforcement for Review Mode and explicit approval handling for supported requests.
- Versioned, transactional SQLite migrations with preventive backups.
- Backup imports parsed in a worker and constrained by file, record, field, and relationship limits.
- Codex App Server recovery, bounded streaming, structured logs, and renderer diagnostics.

## Compatibility

- Packaged target: Linux x64.
- Minimum verified runtime: Node.js 22 LTS for development.
- Minimum Codex CLI: `0.144.0`.
- Verified Codex CLI: `0.144.1`.
- Git is required for repository workflows.
- Pandoc is optional and only required for advanced document exports.

## Validation

The release candidate passed:

- TypeScript type checking;
- ESLint with zero warnings;
- 34 unit and integration tests;
- renderer, main process, and preload production builds;
- a packaged Electron smoke test covering preload exposure, SQLite round-trip, sandboxing, and context isolation.

## Known limitations

- The Codex App Server interface is experimental and can change in newer CLI releases.
- Release artifacts are not code-signed.
- Automatic updates are not implemented.
- Linux x64 is the only packaged and validated platform for this beta.
- PDF and DOCX export quality depends on the local Pandoc toolchain.

## Installation

Download either the AppImage or `.tar.gz` archive from the GitHub release.

```bash
chmod +x "Nocturne Codex-Linux-0.6.0-beta.AppImage"
./"Nocturne Codex-Linux-0.6.0-beta.AppImage"
```

Nocturne Codex uses the authentication of the locally installed Codex CLI. It does not bundle credentials or move project content to a separate application service.
