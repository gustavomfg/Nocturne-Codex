# Nocturne Codex v0.7.0-beta

Release date: July 14, 2026

> Maintenance clarification: the supported security line is `0.7.x-beta`. This release already introduced Electron boundary tests and a packaged runtime smoke; those items are no longer listed as future roadmap work.

`v0.7.0-beta` is the product-polish release of Nocturne Codex. It concentrates on continuity, responsiveness, accessibility, and trustworthy feedback across the complete desktop workflow, turning the existing engineering feature set into a calmer and more predictable daily experience.

## Highlights

- Reading-safe streaming that preserves the user's position and offers an explicit return to the latest response.
- Mutually exclusive compact drawers with keyboard focus containment, Escape dismissal, and focus restoration.
- Quick actions that prepare editable prompts instead of executing ambiguous requests immediately.
- Progressive composer growth for reviewing longer prompts without losing surrounding context.
- Protected workspace-memory edits with discard confirmation, duplicate-save prevention, and visible completion feedback.
- Contextual settings, memory, clipboard, suggestion, and Git feedback presented where the action originated.
- Operationally prioritized Activity and differentiated empty states that explain the next useful action.
- Readiness-driven onboarding and clearer consequences for Build, Review, and Docs modes.
- A consolidated responsive system across five canonical width ranges, with stable typography and control sizing.
- Reliable attachments and artifact deletion even after their originating conversation is removed.

## Experience and accessibility

- Dialogs and compact panels preserve keyboard context and expose urgent failures through assertive live regions.
- Asynchronous controls communicate loading, success, failure, and disabled states without relying on color alone.
- Settings and secondary surfaces avoid expensive full-window visual effects, keeping transitions fluid in packaged builds.
- Interface density, responsive behavior, hover states, and motion are consistent across the application shell.
- The design-system check now recognizes minimum and maximum width declarations reliably.

## Security and reliability

- A single-instance application lock prevents competing processes from operating on the same local database and workspace state.
- SQLite migrations are incremental and transactional, with upgrade coverage from intermediate schema versions.
- Privileged IPC registration is split by capability while preserving shared sender validation, schemas, and canonical limits.
- Large Git diffs are bounded and report truncation instead of exhausting main-process or renderer buffers.
- Codex event routing and buffering are isolated from the application shell, with lifecycle coverage for timeouts, approvals, interruption, crashes, and reconnection.
- Backup restoration, macOS window recreation, and process cleanup use deterministic resource lifecycles.
- Safe Markdown rendering and clipboard boundaries prevent untrusted links or renderer content from gaining unintended authority.
- Electron renderer isolation, sandboxing, typed IPC, sender verification, and workspace confinement remain enforced.
- Persistent data continues to use transactional SQLite migrations and preventive backup snapshots.
- Review Mode remains read-only, and privileged operations continue to require validated IPC and explicit user intent.
- Error feedback remains local to its initiating workflow instead of disappearing into unrelated global state.
- Hardware acceleration is enabled by default, with an explicit `NOCTURNE_DISABLE_GPU=1` fallback for incompatible graphics drivers.
- Streaming, large diffs, activity history, changed files, and Git lists use isolated or bounded rendering paths so long-running sessions do not redraw the whole product or create unbounded DOM trees.
- Long conversations load the newest 100 messages first and retrieve older pages without moving the reader away from the current viewport.
- SQLite uses WAL-appropriate runtime settings, scheduled integrity checks, and safe shutdown optimization.

## Compatibility

- Packaged target: Linux x64.
- Development runtime: Node.js 22.12 or newer within the Node 22 LTS line, with npm 10.
- Minimum Codex CLI: `0.144.0`.
- Verified Codex CLI: `0.144.1`.
- Git is required for repository workflows.
- Pandoc is optional and only required for advanced document exports.

## Validation

The release candidate passed:

- TypeScript type checking;
- ESLint and design-system validation with zero warnings;
- 67 unit and integration tests;
- 21 deterministic renderer scenarios covering keyboard interaction, contextual failures, protected edits, responsive behavior, and large-content stress;
- four visual regression references across desktop and compact widths;
- renderer, main process, and preload production builds;
- a packaged Electron smoke test covering preload exposure, SQLite round-trip, sandboxing, context isolation, and file permissions;
- SHA-256 checksum generation for the published artifacts.

## Known limitations

- The Codex App Server interface is experimental and can change in newer CLI releases.
- Release artifacts are not code-signed.
- Automatic updates are not implemented.
- Linux x64 is the only packaged and validated platform for this beta.
- PDF and DOCX export quality depends on the local Pandoc toolchain.

## Installation

Download either the AppImage or `.tar.gz` archive from the GitHub release.

```bash
chmod +x Nocturne.Codex-Linux-0.7.0-beta.AppImage
./Nocturne.Codex-Linux-0.7.0-beta.AppImage
```

Nocturne Codex uses the authentication of the locally installed Codex CLI. It does not bundle credentials or move project content to a separate application service.
