# Workspace

Cada conversa pertence a uma raiz normalizada. Arquivos anexados, previews e exportações são validados contra essa raiz. Workspaces recentes e favoritos ficam no banco local.

`.nocturne/project.json` descreve stack e comandos; `memory.md` guarda decisões; `rules.md` guarda padrões. `node_modules`, builds, releases, Git, logs e binários são ignorados por padrão durante reviews.
